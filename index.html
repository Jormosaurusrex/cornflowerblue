Introduction

My $deity Why

I am constantly writing a lot of small, self-contained websites, many of which require design and form elements. These projects usually don't need gigantic toolkits, so I would end up writing, re-writing, cutting-and-pasting, and re-using elements from earlier projects. I'm lazy, so I started collecting them.

Of course, once you start down this dark path, forever will it dominate your destiny. Consume you, it will. It's fun for me; I like exploring the weird edges and thinking about what I think are the "best" behaviors for things.

Why package it up? I want to release some of my other code. If cornflowerblue is a dependency for those projects, it should be solid and reusable.

Cornflower blue is a shade of medium-to-light blue containing relatively little green compared to blue. This hue was one of the favorites of the Dutch painter Johannes Vermeer.[1]

The most valuable blue sapphires are called cornflower blue, having a medium-dark violet-blue tone.[2]

Uses

Robert Boyle

Robert Boyle reported a blue dye produced from the cornflower.[3] This was also called Boyle's Blue[4] and Cyan Blue.[5] This dye color however, was not widely commercialized.[6]

X11

Cornflower blue is a defined color in the X Window (X11) color scheme.[7] As such, it is a color available as a named color for webpages.

HTML

CornFlowerBlue is an HTML color name, its hexadecimal code is #6495ED.[8]

Crayola

Cornflower blue is a Crayola color with hexadecimal code #93CCEA[9]. It was originally introduced in 1958, in the box of 48 crayons. The color is also called light cornflower blue.[10]

Microsoft XNA

Cornflower blue is the default clear color used in the XNA framework.

In popular culture

  • Chuck Palahniuk mentions the color at least once in each of his novels. The color also appears in the film adaptation of Palahniuk's Fight Club, where it is associated with the Narrator's boss: "It must've been Tuesday. He was wearing his cornflower-blue tie." In a later scene, the Narrator's boss asks, "Can I get the icon in cornflower blue?"
  • The German popular song "Kornblumenblau" (literally "cornflower blue") humorously glorifies extreme drunkenness, blau being German slang for "drunk" and cornflower blue being an intense shade of the color.[11]
  • The expression is used several times in describing the character Helen Watt, a historian in the book The Weight of Ink by Rachel Kadish.
  • In an episode of the ninth and final season of How I Met Your Mother, Barney Stinson is seen contemplating which tie he should wear for his wedding ceremony and is upset that he does not have his cornflower-blue tie which would have brought out his eyes.
  • In the Witcher novel Blood of Elves by Andrzej Sapkowski, Enchantress Triss Merigold's eyes are described as "cornflower blue."

Message Boxes

Information boxes are simple. The size of the icons changes based on how many lines of text are present.

Standard

new InstructionBox({options});

An InstructionBox is used to display instructions or other information to a user that may be useful or required for filling out a form. A SimpleForm will take instructions as a parameter, so they don't often need to be inserted manually.

Form Response

new MessageBox({options});

When submitted, forms can succeed or fail. In doing so, they can return messages (results, errors, and warnings). Warnings alone do not prevent a form's success. These messages are returned and displayed automagically when using a SimpleForm, so these shouldn't be invoked alone.

A MessageBox is actually a container for multiple information boxes, as a form submission can contain both results and warnings or errors and warnings.

Buttons

new SimpleButton({options});

The basic button class is SimpleButton. They can be set for mute and hot in construction; this will change the button's default state. Any button can be made hot.

Buttons can take an icon parameter. Icons can be placed on either side with iconside (default is left).

Buttons can also be invoked with ConstructiveButton() (used for actions that move the user forward, such as save events) and DestructiveButton(), which is used for destructive actions, such as "cancel".

Normal

Normal buttons have borders and a transparent background in the resting state.

Ghost

Ghost buttons don't have backgrounds in the resting state.

Mute

Mute buttons don't have borders in the resting state.

Link

Buttons with link: true take the behaviors of <a> tags. This is useful when the functionality of a button needs to be styled as a link.

Shaped: Pill

Pill shaped buttons can have text or icons. They can also be ghost or mute.

Shaped: Square

Square shaped buttons only have icons. They can also be ghost or mute.

Shaped: Circle

Circle shaped buttons only have icons. They can also be ghost or mute.

Naked

Buttons with naked: true (or the naked class) have all borders and states reset. This is very similar to link buttons but they retain colors.

Sizes

Buttons can take a size: attribute. Possible values are micro, small, medium (default), large, and fill (which grows to fill its container).

Text Areas

new TextArea({options});

A TextArea is used for large freeform data entry. Text Areas accept "large" data. Short form data entry should be done with various TextInput elements. TextAreas extend InputElement.

Standard

The standard TextArea is configured with counter: sky.

Mute

TextAreas can be made mute, though the displayed border changes.

Passive

TextArea elements can also be made passive, like other inputs.

Disabled

Disabled TextArea elements behave like disabled TextInputs in nearly all respects.

Inputs

new TextInput({options});

A TextInput is one of several types of freeform data entry elements. Text inputs typically accept "short" data - a few words at most - and the data is often specifically typed (long form text entry should be accomplished with a TextArea element). All TextInput classes extend InputElement.

Standard

TextInputs all inherit many behaviors straight out of the box, such as messages and warnings. Each supports built-in data validators, and all may be marked essential with required: true.

If given a help parameter (accepts a string), a help button will be included with the label. It behaves as a tooltip on hover, can be locked open on click, and will automatically appear after a short period of the element retains focus but has no activity.

TextInput supports three kinds of character counters, enabled with counter: type, were type can be one of 'sky', 'remaining', or 'limit'. This changes the way the counter is worded.

Passive

Inputs can be switched to an "passive" mode, which displays their labels and values but without allowing for interaction or alteration (though they can still be updated programmatically). This serves a different purpose than disabled.

The passive status can be defined in its construction (with passive: true) or toggled programmatically (with .deactivate(), .activate(), and .toggleActivation()).

Mute

Setting an input to with mute: true will change its display style. The label changes position and top controls disappear (such as character counters) fade until the element obtains focus.

Password

new PasswordInput({options});

The PasswordInput element is for redacted content. By default, these elements include a password visibility toggle. It can be turned off by setting visibilityswitch to false. Set startvisible to true to have it begin with the password unredacted.

File

new FileInput({options});

The FileInput element is for uploading files of any type.

By default, it automatically accepts image types, but the accept: parameter will change that.

Including a FileInput element in a SimpleForm will automatically adjust the form's encapsulation type.

Number

new NumberInput({options});

The input accepts non-numeric values but treats all of them as errors. It will also fire validators that test against minnumber or maxnumber, and are divisible by any value given to step. If wholenumbers is true it will not accept decimals. The steppers can be turned off with steppers: false. If the input has focus, up and down arrows will change the values.

The input accepts non-numeric values but treats all of them as errors.

Email

new EmailInput({options});

The email input has an email address validator that is enabled by default but can be disabled with forceconstraints: false.

URI

new URIInput({options});

The URI input has a validator that is enabled by default but can be disabled with forceconstraints: false.

Disabled

An entire element can be set to be disabled with disabled: true. If the element has been provided with a help entry, it will still be reachable.

Hidden

new HiddenInput({options});

A HiddenInput is just that: hidden. It will not display to the user, nor will it appear in screen readers. Do not use a HiddenInput if the input may ever become visible.

Selects

A SelectMenu is an enumerated list of options from which only one item may be selected and replaces the standard html <select> element. Use a SelectMenu when you have more 5 or more options; otherwise, consider using a RadioGroup.

This component allows for great keyboard navigation and searching and is smart about the direction it opens, doing its best to not open off-screen.

Under the hood, the SelectMenu is implemented with Radio Buttons so it maintains compatibility.

Standard

new SelectMenu({options});

Options for the SelectMenu list are provided with the options: parameter. This is an array of option definitions, which look like { label: "Label to show", value: "v" }. Include checked: true to automatically select that option.

Use the unselectedtext option to set the "non" option. Set to null to prevent the inclusion of an unselected element. Giving it a prefix will attach that before options when they're selected.

By default, a small search indicator will appear if the user begins typing while the input has focus. Set searchtext to false to disable this. The display will switch between top and bottom, depending on which direction the menu opens.

Passive

SelectMenu elements can also be made passive, like other inputs.

Mute

SelectMenu elements can also be made mute, like other inputs.

Note that the label is always in the "focused" position, as SelectMenu elements always have a value, even if that value is "unset".

Disabled

SelectMenu elements can also be made disabled, like other inputs.

State Menus

new StateMenu({options});

A StateMenu is a pre-purposed SelectMenu. It supports both the United States and Canada. By default, all the states are included in the option list, but it can take a filter option to restrict to a single country (values of us or ca).

By default, the value provided by the menu will be the capitalized 2 letter code for the state. This can be changed with valuesas (takes "code" or "name") to change.

By default, the value provided by the menu will be the capitalized 2 letter code for the state. This can be changed with valuesas (takes "code" or "name") to change.

Radio Buttons

new RadioGroup({options});

A RadioGroup is similar to a SelectMenu except that all of the options are visible to the user from the outset. This is desirable when comparison is important, but the largest benefit is the reduction of the cognative load on the user (they don't have to click open the menu nor do they have to scroll the list).

If you have 5 more options, consider using a SelectMenu. If you only have two options, consider whether a BooleanToggle is a better solution.

Standard

The standard mode of a RadioGroup inherits most of the behaviors of other inputs, including the help system. While they can be marked required (valid only if they start out without a selected entry), there aren't many other errors that this kind of enumeration can create.

There really can't be a mute variant of a RadioGroup.

Passive

As with other inputs, a RadioGroup can be made passive.

The passive status can be defined in its construction (with passive: true) or toggled programmatically (with .pacify(), .activate(), and .toggleActivation()).

Disabled

RadioGroup elements can also be made disabled, like other inputs.

Checkboxes and Toggles

new BooleanToggle({options});

A BooleanToggle is a switch that can be in one of two states (checked/unchecked, on/off, true/false). It is an implementation of <input type='checkbox' />.

BooleanToggles can take several display styles. Some are better suited for specific purposes than others. For instance, if the state of the value it controls is important, a toggle style might be a better choice than a simple checkbox.

Default

Labels can be on either side. The default is right-sided, because the affordances end up in line.

Styled

Styles can be applied to the toggles. Values for the style attribute include square, circle, switch, and toggle.

Forms

new SimpleForm({options});

The SimpleForm class is the form management system. SimpleForm has built-in validation, submission, and response handling systems.

Forms are displayed with discrete sections: header, elements, and actions. The header section will include provided instructions (within an included InstructionBox) and any MessageBox elements generated by the form submission.

Buttons passed with submits: true will submit the form for processing when clicked. This also will be enabled when the form passes validation.

Standard

Profile

Forms can be made passive, like inputs. Forms that can be made passive accepts passiveinstructions and passiveactions parameters, both of which displays when the form is in passive mode. If either are omitted, no harm and no foul.

Dialogs

new DialogWindow({options});

The DialogWindow class can display a simple title (with title:) or define its own header (with header, which takes a jQuery object to display). Dialog content is provided with one of two attributes: content (which takes a jQuery object) or form (which takes a SimpleForm).

If passed a form, the dialog will render the form's header, elements, and actions in a way that compliments the dialog.

If passed a jQuery object with content, no special formatting is applied; the content must be styled on its own. This can be passed a selector (and thus drawing content from the existing DOM), but it will remove that content from wherever it was and destroy it when the window is closed, so use .clone().

Growlers

new Growler({options});

A Growler is a short-lived, auto-dismissable notifications that presents a simple payloads.

They can be targeted to open in any one of six positions. They can be configured to show title or text payload individually. Supplied icons will be included in the title if no payload is provided.

It is possible to set them to require manual dismissal by setting the duration to 0 or -1.

Positions

Special Types

Growlers can be invoked with shorthand static methods. These growlers are lightweight and pre-configured.

Growler.growl('This is a growl message!', 'Growler.growl');
Growler.success('This is a success message!');
Growler.error('This is an error message!');
Growler.warn('This is an warning message!');
Growler.caution('This is an caution message!');

Tabs and Menus

Button Menu

new ButtonMenu({options});

A ButtonMenu is just that: a SimpleButton that opens a menu. Accordingly, it accepts all SimpleButton parameters, with the addition of secondicon (for the menu icon) and then an items array for each menu entry.

The items array contains a dictionary of parameters: label, icon (optional), and action, which is a function.

Tab Bar

new TabBar({options});

A TabBar is basically a permanently open ButtonMenu. It is mostly intended for navigation.

It can also take a vertical: true parameter, which is better for sidebar menus (like this demo!).

Password Changer

Simple

new PasswordChangeForm({options});

A PasswordChangeForm is a self-contained password manipulation form. It can be passed an array of values that it will reject as acceptable with the cannotbe: parameter and has a hook for checking passwords against external bad password lists (like NIST) using badpasswordhook:.

It includes a PasswordGenerator component, which can be used anywhere else. The generator can be configured by the user to include various sets of characters in the generation.

Progress Meters

Simple

new SimpleProgressMeter({options});

The SimpleProgressMeter component is designed to display one point of data along a single axis. By default it, it assumes a scale between 0 and 100.

If given previousrank or nextrank parameters, it will grow taller and display them inside of the bar, a long with the scale values (which can be disabled with showcaps: false).

Adding the pillcap class makes the endpoint circular, but this looks weird if the bar is taller.

Radial

new RadialProgressMeter({options});

A RadialProgressMeter is just that: circular. This is implemented using in-line SVG objects and goofing around with the border dash attributes.

Since they are implemented in SVG they cannot be easily sized via css. They take a size parameter which can be a string ('small', 'medium', 'large', 'huge'), values in 'px' or 'em', or a number (which gets interpreted as pixels). The default is medium.

There are two basic style values: solid (the default) and ticks, which causes the meter do display as fine lines.
